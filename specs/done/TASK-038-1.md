# TASK-038-1 - 异步重构（async stream_run + 后端自动 finalize）

## 目标
将 stream_run 链路改为 async generator，使客户端断开时后端立刻感知并自动落库，无需前端主动调 finalize 接口。

## 产品层
Runtime / Infrastructure

## 背景
TASK-037 完成了文字流式阶段的 Stop + 截断保存，但：
1. 工具执行阶段是同步阻塞，Stop 后工具仍会跑完（响应慢）
2. finalize 落库依赖前端网络请求成功（不可靠）

改为 async 后，CancelledError 在工具执行阶段也能触发，后端自动落库，前端只需断开连接。

## 用户动作
用户在 agent 任意阶段（文字流或工具执行中）点击 Stop。

## 用户会看到
- Stop 响应更快（不再等工具跑完）
- 截断内容自动保存，上下文连续
- 前端不需要主动调 finalize（内部简化）

## 范围内
- `model/openai_adapter.py`：`stream_generate` 改为 `async def` + `async for`
- `runtime/agent_runtime.py`：`stream_run` 改为 `async def`；`handle_tool_calls` 改为 `async def`
- `application/run_service.py`：`stream_agent_service` 改为 `async def`；加 `try/except asyncio.CancelledError` 自动调 finalize 落库
- `api/routes/run_routes.py`：StreamingResponse 改用 async generator
- 幂等保护：run_id 唯一约束，重复 finalize 静默忽略
- 单元测试同步更新

## 范围外
- Run 状态机（status 字段）→ TASK-038-2
- Tool call 独立记录表 → TASK-038-2
- 工具超时机制 → TASK-038-2
- DB migration → TASK-038-2
- 非流式 run_agent_service（保持同步）
- 前端改动（前端 Stop 逻辑已在 TASK-037 前端部分完成）

## 实现步骤
1. `openai_adapter.py`：`stream_generate` 改 async
2. `agent_runtime.py`：`handle_tool_calls` 改 async；`stream_run` 改 async generator
3. `run_service.py`：`stream_agent_service` 改 async；加 `try/except asyncio.CancelledError` 自动落库
4. `run_routes.py`：StreamingResponse 使用 async generator
5. 幂等保护：save_partial_run 检查 run_id 是否已存在
6. 更新受影响的单元测试
7. 全量测试通过

## 完成标准
- 工具执行阶段 Stop → 工具立刻中断 → 后端自动落库 → 上下文连续
- 文字流阶段 Stop → 后端自动落库（不依赖前端 finalize 调用）
- 全量测试通过
- 非流式 `/run` 不受影响
