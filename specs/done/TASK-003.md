# TASK-003

## Goal
把当前基于内存的 `session_store` 提升为可持久化的会话存储，让 agent 服务重启后仍能恢复指定 `session_id` 的上下文。

## In Scope
- 抽象出会话读、写、删的存储接口
- 保留现有 `session_id` 隔离语义
- 实现一个最小可用的持久化方案，优先单机单进程可运行
- 服务重启后可恢复已有会话
- 保留 `/reset` 的清空能力
- 更新测试，覆盖持久化恢复和 reset 行为

## Out of Scope
- 多机共享会话
- 分布式锁
- 鉴权与权限系统
- 多 Agent 编排
- 复杂过期策略

## Done when
- 同一 `session_id` 的历史状态能写入持久层并在重启后恢复
- 不同 `session_id` 仍然彼此隔离
- `reset` 能清空指定会话并重新开始
- 存储层接口和实现分层清晰，后续可以替换底层实现
