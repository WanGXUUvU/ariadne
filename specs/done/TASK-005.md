# TASK-005

## Goal
把当前 agent 的执行过程整理成稳定、可追踪的结构化输出，让 `/run` 既保留最终回复，也能清楚展示工具调用和执行轨迹，方便后续调试和前端展示。

## In Scope
- 明确 `AgentEvent` 的事件类型和字段边界
- 保证一次请求里的事件顺序稳定可读
- 让工具调用、工具结果、最终回复都能被结构化记录
- 保持现有 `session_id` 隔离和 SQLite 持久化行为不变
- 更新测试，覆盖纯回复和工具调用两条主流程

## Out of Scope
- 真正的流式传输
- 前端 UI 展示
- 多工具并行调度
- 多 Agent 编排
- RAG

## Done when
- `/run` 能返回稳定的结构化执行轨迹
- 工具调用链路仍然能正常跑通
- 用户最终看到的回复不被内部轨迹污染
- 现有会话持久化与迁移链路不受影响
